Crazyflie Nano UAV Onboard SLAM

This project was a semester‑long design and build of an autonomous rover capable of navigating a walled maze, localizing from an arbitrary start pose, and retrieving and delivering a block to a designated drop‑off zone within a five‑minute time limit.

The rover used a compact Kiwi drivetrain for holonomic motion, a top‑mounted 2D LiDAR for 360° environmental scans, and a retractable gripper arm that kept the footprint small enough for tight maze navigation The software stack was built from the ground up in C++ on an STM32F4 microcontroller, with a custom RTOS scheduler to manage concurrent tasks. The SLAM implementation was a particle filter with a sparse pose graph for loop closure, optimized for the limited compute resources. The system was tested and refined through multiple iterations in a hardware‑in‑the‑loop simulator before final deployment on the physical rover.

Technical Architecture

System overview

Drivetrain: Three‑wheel Kiwi drive for omnidirectional motion and a small turning radius, simplifying navigation around 1’×1’ maze cells and tight corners Sensing: A 2D LiDAR provided dense 360° scans for localization and manual obstacle avoidance; ground‑level ultrasonic sensors were installed but not fully integrated by the final milestone Compute: An Arduino Uno handled low‑level drivetrain control, while an Arduino Mega handled Bluetooth teleoperation and sensor I/O, separating timing‑critical motion control from higher‑level communication and perception Manipulator: A retractable arm with a simple parallel gripper lifted the block with about 2 mm of clearance and had minimal impact on stability and chassis size

What I Learned

Key Lessons

  • Plan power distribution and grounding early; fragile power architecture can mask other issues and constrain integration and testing time.
  • Design and tuning of holonomic drive kinematics and PID control on resource‑limited microcontrollers.
  • The value of modular architecture and clear interfaces between drivetrain, sensing, UI, and manipulation in multi‑subsystem mechatronics projects.
